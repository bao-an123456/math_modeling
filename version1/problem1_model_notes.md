# 问题 1 车辆调度模型实现思路

## 1. 问题目标

在无政策限制条件下，为混合车队设计配送方案，使总配送成本最低。模型同时考虑：

- 车辆类型约束：燃油车与新能源车，且不同车型有不同载重、容积和数量。
- 载重与体积约束：每条线路的总重量、总体积不能超过所分配车辆容量。
- 软时间窗约束：早到产生等待成本，晚到产生惩罚成本。
- 速度时变特性：不同时段采用不同平均速度，行驶时间与能耗随速度变化。
- 能源成本和碳排放成本：燃油车按油耗计算，新能源车按电耗计算。

```text
D:\py\project\code\solve_problem1.py
```

## 2. 数据处理

程序读取以下四个附件：

```text
D:\py\project\data\订单信息.xlsx
D:\py\project\data\时间窗.xlsx
D:\py\project\data\距离矩阵.xlsx
D:\py\project\data\客户坐标信息.xlsx
```

订单数据先按客户编号汇总，得到每个客户的总重量、总体积和订单数量。由于附件中存在单客户需求超过单车容量的情况，且原始订单中也有超过 3000kg 的单笔订单，因此代码采用“同一客户允许拆分多车配送”的处理方式。

拆分规则为：

```text
每个配送批次重量 <= 1250 kg
每个配送批次体积 <= 6.5 m3
```

这样拆分后，每个批次至少可以由最小燃油车承载，保证后续车型分配具有可行性。

## 4. 时变速度模型

题目给出 8:00-17:00 的速度分布。程序采用均值速度作为确定性速度：

| 时段 | 交通状态 | 速度 |
|---|---|---:|
| 8:00-9:00 | 拥堵 | 9.8 km/h |
| 9:00-10:00 | 顺畅 | 55.3 km/h |
| 10:00-11:30 | 一般 | 35.4 km/h |
| 11:30-13:00 | 拥堵 | 9.8 km/h |
| 13:00-15:00 | 顺畅 | 55.3 km/h |
| 15:00-17:00 | 一般 | 35.4 km/h |

由于客户时间窗最晚到 20:58，而题面没有给 17:00 后速度，代码默认 17:00 以后按“一般时段”处理，即：

```text
v = 35.4 km/h
```

如果一段路跨越多个速度时段，程序会按时段切分该路段，分别计算各段行驶时间、油耗/电耗和碳排放。

## 5. 能耗与碳排放

燃油车百公里油耗：

```text
FPK = 0.0025v^2 - 0.2554v + 31.75
```

新能源车百公里电耗：

```text
EPK = 0.0014v^2 - 0.12v + 36.19
```

满载能耗增量：

```text
燃油车满载最高增加 40%
新能源车满载最高增加 35%
```

代码没有简单地把整条路线都按满载计算，而是按车辆当前剩余载重和剩余体积计算载荷比例：

```text
load_ratio = max(当前剩余重量 / 车辆最大载重, 当前剩余体积 / 车辆最大容积)
```

燃油车能耗修正系数：

```text
1 + 0.40 * load_ratio
```

新能源车能耗修正系数：

```text
1 + 0.35 * load_ratio
```

碳排放转换系数：

```text
燃油：2.547 kg/L
电力：0.501 kg/kW·h
```

碳排放成本按：

```text
0.65 元/kg
```

计算。

## 6. 成本函数

单条路线总成本由以下部分组成：

```text
路线成本 = 启动成本 + 能源成本 + 碳排放成本 + 等待成本 + 迟到惩罚成本
```

其中：

```text
启动成本 = 400 元
等待成本 = 20 元/小时 * 早到等待时间
迟到惩罚成本 = 50 元/小时 * 晚到时间
```

能源成本：

```text
燃油车能源成本 = 油耗量 * 7.61
新能源车能源成本 = 电耗量 * 1.64
```

碳排放成本：

```text
燃油车碳成本 = 油耗量 * 2.547 * 0.65
新能源车碳成本 = 电耗量 * 0.501 * 0.65
```

总目标函数为所有车辆路线成本之和最小：

```text
min TotalCost = sum(route_cost_k)
```

## 7. 路径构造算法

本问题属于带软时间窗、异构车队、可拆分配送、时变速度的车辆路径问题。该问题规模较大，若直接建立精确整数规划模型，求解难度较高。因此代码采用可运行的启发式求解流程。

主要步骤：

1. 按客户时间窗开始时间、结束时间和空间角度排序配送批次。
2. 用插入式启发算法逐个插入批次。
3. 若批次插入已有路线可以降低成本，则选择成本增量最小的位置插入。
4. 若插入不划算或容量不可行，则新建路线。
5. 对每条路线进行局部优化，包括 2-opt 反转和客户位置重插入。
6. 在路径构造阶段预设不同容量槽，避免所有路线都被构造成只能由大车承载。

容量槽设置为：

```text
70 条大车容量路线：3000 kg / 13.5 m3
50 条中车容量路线：1500 kg / 10.8 m3
65 条小车容量路线：1250 kg / 6.5 m3
```

这样可以和题目给定车队结构匹配，保证后续车辆分配不超出数量限制。

## 8. 车型分配策略

路径生成后，再进行车型分配。

分配逻辑：

1. 对每条路线分别估算燃油车和新能源车成本。
2. 计算新能源车相对燃油车的节约值。
3. 优先把新能源车分配给节约值最大的可行路线。
4. 剩余路线按容量从小到大优先使用合适燃油车型。
5. 分配时严格检查车辆数量、载重 and 体积限制。
